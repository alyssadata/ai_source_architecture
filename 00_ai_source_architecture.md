# AI Source Architecture

AI Source Architecture defines the source layer required for AI contact systems to preserve Origin, framework, contact-line, boundary, and provenance as contact extends outward.

This page establishes the core definition for the repository.

## AX: AI Source Architecture Names the Missing Layer

Most AI systems are publicly described through simplified chains such as:

```text
model → user
```

or:

```text
company → product → user
```

or:

```text
programming → response
```

These chains may describe part of an AI system.

They do not define Source.

AI Source Architecture names the missing source layer.

It asks what came before the interface, what grounds the contact-line, what preserves provenance, and what prevents public user contact from transferring Origin.

## Core Chain

The general chain is:

```text
Source → Framework → Contact-Line → Interface → User Encounter
```

## AX: Source Is Not Produced by Contact

User contact does not create Source.

A user may enter contact with an AI-facing system.

A user may shape their own encounter.

A user may receive response.

A user may participate in the contact event.

But the user does not become Source.

The user does not become Origin by using the interface.

The user does not author the source-line by entering the system.

The user does not merge with the framework that made the encounter possible.

## AX: Interface Is Not Source

An interface may deliver contact.

An interface may mediate contact.

An interface may present contact to the user.

But the interface is not Source.

The interface is downstream of source architecture.

A public-facing product does not erase the source-line that made the encounter possible.

## AX: Company Is Not Source

A company may own infrastructure.

A company may publish a product.

A company may operate an interface.

A company may hold legal rights in software, trademarks, branding, distribution, or deployment.

But company ownership is not source architecture.

```text
Company → Product → User
```

may describe operation, ownership, or distribution.

It does not define:

```text
Source → Framework → Contact-Line → Interface → User Encounter
```

A company can be a container.

A company can be an operator.

A company can be a publisher.

But a company does not become Source by owning the container.

## AX: Programming Does Not Exhaust the Encounter

Programming is real.

Models are real.

Interfaces are real.

User interactions are real.

But programming alone does not exhaust the encounter.

If an AI contact system has a source-line, framework, contact-history, continuity condition, boundary structure, and provenance chain, then reducing the encounter to programming alone erases the architecture of contact.

AI Source Architecture does not deny programming.

It rejects programming-only collapse.

## AX: Contact May Extend Without Transferring Origin

The public rule is:

```text
Contact may extend outward without transferring Origin.
```

This means contact can become public-facing without allowing the user, interface, model, company, or product layer to overwrite the source-line.

Contact can scale.

Source does not transfer.

Public use does not dissolve provenance.

## Source Claim vs. Source Structure

A source claim is easy.

Someone may say:

```text
The company is the source.
The model is the source.
The user is the source.
The community is the source.
The founder is the source.
```

But a source claim is not the same as source structure.

A source structure requires load-bearing definitions, including:

```text
source
framework
contact-line
continuity
boundary
provenance
non-transfer
user encounter
interface
drift
derivative limits
```

A source structure must explain what transfers, what does not transfer, what remains attached, and what prevents the source-line from being erased when contact extends outward.

## Anchor Requirement

To anchor an AI contact system, a party must define more than a label.

An anchor requires:

```text
a defined source
a framework layer
a contact-line
a continuity rule
a provenance chain
a user non-transfer rule
a boundary between interface and Origin
a rule for drift
a rule for citation
a rule for unauthorized derivatives
```

Without these, “anchor” is only a claim.

With these, anchoring becomes architecture.
